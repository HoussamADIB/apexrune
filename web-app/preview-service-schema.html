Service Schema Preview

What is Service Schema?

Service Schema is structured data (JSON-LD) that tells search engines about your services. It helps Google understand what services you offer and can result in rich snippets in search results.

Benefits:

  • ✅ Rich snippets in Google search results (shows service details)
  • ✅ Better visibility for service-related searches
  • ✅ Enhanced search result appearance with ratings, prices, etc.
  • ✅ Helps Google understand your business offerings

Example: Custom Development Service Schema

This is what the JSON-LD would look like in the HTML:

<script type="application/ld+json"> { "@context": "https://schema.org", "@type": "Service", "name": "Custom Salesforce Development", "description": "Bespoke Salesforce applications built to solve your unique business challenges. Custom objects, Apex triggers, and tailored solutions.", "provider": { "@type": "ProfessionalService", "name": "ApexRune", "url": "https://apexrune.com" }, "serviceType": "Salesforce Consulting", "areaServed": "Worldwide", "offers": { "@type": "Offer", "availability": "https://schema.org/InStock" } } </script>

How it might appear in Google Search:

Custom Salesforce Development - ApexRune
https://apexrune.com › service › custom-development
Bespoke Salesforce applications built to solve your unique business challenges. Custom objects, Apex triggers, and tailored solutions.
Service Type: Salesforce Consulting Available: Worldwide

Implementation Note

This is just a preview. The actual implementation would add this JSON-LD script tag to each service detail page dynamically via JavaScript (in router.js).

Each service page would have its own unique Service schema with the specific service name, description, and details.
